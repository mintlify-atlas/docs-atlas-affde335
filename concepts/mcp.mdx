MCP (Model Context Protocol) servers extend agent capabilities by providing access to external systems, documentation, and tools. hatch3r integrates 5 MCP servers with optional additional servers for specific use cases.

## MCP Architecture

hatch3r's MCP configuration lives in `/.agents/mcp/mcp.json` (canonical source). During `hatch3r init`, this configuration is adapted to each tool:

- **Cursor**: `.cursor/mcp.json` or project root `mcp.json`
- **Claude Code**: `.mcp.json` in project root
- **VS Code / Copilot**: `.vscode/mcp.json`
- **Cline / Roo**: `.roo/mcp.json`

### Secrets Management

`hatch3r init` creates a `.env.mcp` file at the project root with all required environment variables:

```bash
# .env.mcp (generated by hatch3r init)
GITHUB_PAT=ghp_xxxx
BRAVE_API_KEY=xxxx
```

This file is gitignored by default.

**Secret loading by editor**:

- **VS Code / Copilot**: Secrets load automatically from `.env.mcp` via the native `envFile` field
- **Cursor / Claude Code / others**: Source the file before launching: `set -a && source .env.mcp && set +a && cursor .`

See the [MCP Setup Guide](/guides/mcp-setup) for full details and PAT scope guidance.

## The 5 Core MCP Servers

<AccordionGroup>
  <Accordion title="GitHub" icon="github">
    **Provider**: GitHub Copilot MCP API
    
    **Capabilities**:
    - Repository management (issues, PRs, code search)
    - Project boards (Projects V2 GraphQL operations)
    - Code review and comments
    - Labels, milestones, releases
    
    **Configuration**:
    ```json
    {
      "github": {
        "url": "https://api.githubcopilot.com/mcp/",
        "headers": {
          "Authorization": "Bearer ${env:GITHUB_PAT}",
          "X-MCP-Toolsets": "all"
        }
      }
    }
    ```
    
    **Required secret**: `GITHUB_PAT` (Personal Access Token)
    
    **PAT scopes**: `repo`, `workflow`, `project` (read/write)
    
    **Tooling hierarchy**: Use `gh` CLI first, fallback to GitHub MCP for Projects V2 mutations not covered by `gh`
    
    **Used by**: All board commands (`board-init`, `board-fill`, `board-pickup`, `board-refresh`), planning commands, issue-workflow
  </Accordion>
  
  <Accordion title="Context7" icon="book">
    **Provider**: Upstash Context7
    
    **Capabilities**:
    - Up-to-date, version-specific library documentation
    - Resolve library IDs (`resolve-library-id`)
    - Query documentation (`query-docs`)
    - Supports 1000+ popular libraries (React, Next.js, TypeScript, etc.)
    
    **Configuration**:
    ```json
    {
      "context7": {
        "command": "npx",
        "args": ["-y", "@upstash/context7-mcp@2.1.1"]
      }
    }
    ```
    
    **No secrets required** (uses public documentation sources)
    
    **Tooling hierarchy**: Tier 3 (after project docs and codebase exploration, before web research)
    
    **Used by**: `hatch3r-researcher` (library-docs mode), `hatch3r-implementer` for external API lookups
    
    **Example usage**:
    ```typescript
    // 1. Resolve library ID
    const libId = await context7.resolveLibraryId('react');
    // 2. Query specific API
    const docs = await context7.queryDocs(libId, 'useState hook');
    ```
  </Accordion>
  
  <Accordion title="Filesystem" icon="folder">
    **Provider**: Model Context Protocol
    
    **Capabilities**:
    - File management (create, read, update, delete)
    - Directory operations
    - File search and globbing
    - Code editing operations
    
    **Configuration**:
    ```json
    {
      "filesystem": {
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-filesystem@2026.1.14", "."]
      }
    }
    ```
    
    **Scope**: Current directory (`.`)
    
    **Used by**: All agents for reading/writing code, specs, docs
    
    **Note**: hatch3r agents prefer dedicated Read/Write/Edit tools over MCP filesystem operations for better user experience
  </Accordion>
  
  <Accordion title="Playwright" icon="browser">
    **Provider**: Playwright MCP Server
    
    **Capabilities**:
    - Browser automation (Chromium, Firefox, WebKit)
    - Web testing and UI interaction
    - Screenshot capture
    - Network interception
    - Console log monitoring
    
    **Configuration**:
    ```json
    {
      "playwright": {
        "command": "npx",
        "args": ["-y", "@playwright/mcp@0.0.68"]
      }
    }
    ```
    
    **No secrets required**
    
    **Used by**: `hatch3r-implementer` for browser verification (Step 6b), `hatch3r-a11y-auditor` for accessibility testing
    
    **Workflow** (from browser-verification rule):
    1. Ensure dev server is running
    2. Navigate to affected page
    3. Interact with changed elements
    4. Visual regression checks
    5. Screenshot evidence for PRs
    6. Browser console error checks
  </Accordion>
  
  <Accordion title="Brave Search" icon="magnifying-glass">
    **Provider**: Model Context Protocol
    
    **Capabilities**:
    - Web research and fact-checking
    - Current information retrieval
    - CVE lookups
    - Breaking change discovery
    - Best practices research
    
    **Configuration**:
    ```json
    {
      "brave-search": {
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-brave-search@0.6.2"],
        "env": {
          "BRAVE_API_KEY": "${env:BRAVE_API_KEY}"
        }
      }
    }
    ```
    
    **Required secret**: `BRAVE_API_KEY` (free tier: 2000 queries/month)
    
    **Get API key**: [brave.com/search/api](https://brave.com/search/api)
    
    **Tooling hierarchy**: Tier 4 (last resort after project docs, codebase, Context7)
    
    **Used by**: `hatch3r-researcher` (prior-art mode), all agents for novel problems, security advisories, current best practices
  </Accordion>
</AccordionGroup>

## Optional MCP Servers

hatch3r's canonical MCP config includes these servers disabled by default. Enable by removing `"_disabled": true`:

<AccordionGroup>
  <Accordion title="Sentry" icon="bug">
    **Purpose**: Error tracking and performance monitoring
    
    **Configuration**:
    ```json
    {
      "sentry": {
        "command": "npx",
        "args": ["-y", "@sentry/mcp-server@latest"],
        "env": {
          "SENTRY_AUTH_TOKEN": "${env:SENTRY_AUTH_TOKEN}"
        }
      }
    }
    ```
    
    **When to enable**: For incident-response skill, production debugging
  </Accordion>
  
  <Accordion title="Postgres" icon="database">
    **Purpose**: PostgreSQL database queries and schema inspection
    
    **Configuration**:
    ```json
    {
      "postgres": {
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-postgres"],
        "env": {
          "POSTGRES_URL": "${env:POSTGRES_URL}"
        }
      }
    }
    ```
    
    **When to enable**: For database migrations, schema changes, query optimization
  </Accordion>
  
  <Accordion title="Linear" icon="list-check">
    **Purpose**: Linear issue tracking and project management
    
    **Configuration**:
    ```json
    {
      "linear": {
        "command": "npx",
        "args": ["-y", "@mkusaka/mcp-server-linear"],
        "env": {
          "LINEAR_API_KEY": "${env:LINEAR_API_KEY}"
        }
      }
    }
    ```
    
    **When to enable**: If using Linear instead of GitHub Issues (note: board commands require GitHub)
  </Accordion>
</AccordionGroup>

## Tooling Hierarchy

The `hatch3r-tooling-hierarchy` rule (scope: always) enforces priority order for knowledge sources:

1. **Project docs**: `docs/specs/`, `docs/adr/`, `README.md` (canonical truth)
2. **Codebase exploration**: Grep, glob, read source code
3. **Library docs**: Context7 MCP (`resolve-library-id` → `query-docs`)
4. **Web research**: Brave Search MCP for current info, CVEs, breaking changes

**GitHub CLI-first**: Use `gh` CLI over GitHub MCP for issues, PRs, code search. Fallback to GitHub MCP only for Projects V2 mutations.

**Browser verification**: Use Playwright MCP for UI changes following the browser-verification rule.

## MCP Configuration in hatch.json

The board commands read GitHub context from `hatch.json`:

```json
{
  "board": {
    "owner": "my-org",
    "repo": "my-repo",
    "projectNumber": 1,
    "defaultBranch": "main",
    "statusOptions": {
      "backlog": "Backlog",
      "ready": "Ready",
      "inProgress": "In Progress",
      "inReview": "In Review",
      "done": "Done"
    },
    "areas": [
      "area:frontend",
      "area:backend",
      "area:infra"
    ]
  }
}
```

This config is auto-populated by `board-init`.

## MCP Server Selection Guide

| Use Case | Required MCP Servers |
|----------|---------------------|
| **Greenfield project** | GitHub, Context7, Filesystem, Brave Search |
| **Brownfield project** | GitHub, Context7, Filesystem, Brave Search |
| **UI-heavy project** | Add Playwright for browser verification |
| **Database-heavy project** | Add Postgres for schema operations |
| **Production monitoring** | Add Sentry for incident response |
| **Linear users** | Add Linear (note: board commands still require GitHub) |

## Performance Considerations

### Token Usage

- **Context7**: Low token cost (returns only relevant docs)
- **Brave Search**: Medium token cost (returns search results)
- **Playwright**: High token cost (browser automation logs)
- **GitHub**: Medium token cost (API responses)

### Rate Limits

- **GitHub PAT**: 5000 requests/hour for authenticated requests
- **Brave Search**: 2000 queries/month (free tier), 20K/month (paid)
- **Context7**: No rate limits (uses cached documentation)
- **Playwright**: No rate limits (local execution)

## Troubleshooting

<AccordionGroup>
  <Accordion title="GitHub MCP not connecting">
    **Symptoms**: `401 Unauthorized` errors
    
    **Solutions**:
    1. Verify `GITHUB_PAT` is set in `.env.mcp`
    2. Check PAT has required scopes (`repo`, `workflow`, `project`)
    3. Confirm PAT hasn't expired
    4. Source `.env.mcp` before launching editor (non-VS Code)
  </Accordion>
  
  <Accordion title="Context7 returning empty results">
    **Symptoms**: "No documentation found"
    
    **Solutions**:
    1. Verify library name spelling (use `resolve-library-id` first)
    2. Check library is in Context7's supported list
    3. Try more specific query ("useState hook" not just "useState")
    4. Fallback to web search if library not supported
  </Accordion>
  
  <Accordion title="Playwright browser not launching">
    **Symptoms**: "Browser executable not found"
    
    **Solutions**:
    1. Run `npx playwright install` to download browsers
    2. Ensure sufficient disk space
    3. Check no conflicting Playwright installations
  </Accordion>
  
  <Accordion title="Brave Search quota exceeded">
    **Symptoms**: "Rate limit exceeded"
    
    **Solutions**:
    1. Wait until monthly quota resets
    2. Upgrade to paid tier (20K queries/month)
    3. Use web search more selectively (only for novel problems)
  </Accordion>
</AccordionGroup>
