## Overview

The `hatch3r add` command will allow you to install community-contributed packs that extend your agent setup with specialized configurations, agents, skills, and rules.

<Warning>
  This command is not yet implemented. It's planned for a future release.
</Warning>

## Planned Usage

```bash
hatch3r add [pack]
```

## What It Will Do

When implemented, the add command will:

1. **Browse community packs** - If no pack specified, show available packs
2. **Install pack** - Download and integrate pack into your `.agents/` directory
3. **Merge configurations** - Safely merge pack content with existing setup
4. **Update manifest** - Track installed packs in `.agents/hatch.json`
5. **Sync outputs** - Generate tool-specific configs for new content

## Expected Pack Structure

Community packs will likely include:

```
pack-name/
├── pack.json           # Pack metadata
├── agents/             # Custom agents
├── skills/             # Skills
├── rules/              # Rules
├── prompts/            # Prompts
├── commands/           # Commands
└── README.md           # Documentation
```

## Planned Installation Process

1. **Download pack** from registry
2. **Validate compatibility** with current setup
3. **Show preview** of changes
4. **Confirm installation**
5. **Merge content** into `.agents/`
6. **Update manifest** to track pack
7. **Run sync** to generate outputs
8. **Validate** final state

## Example Use Cases

### New Project Setup

```bash

hatch3r init

# Add framework pack
hatch3r add nextjs

# Add testing pack
hatch3r add testing-pro
```

```bash
# Current setup
hatch3r status

hatch3r add security-audit

hatch3r validate
```

## Alternative: Manual Installation

Until `add` is available, install community content manually:

```bash
# Clone pack repo
git clone https://github.com/community/typescript-pack

# Copy to .agents/
cp -r typescript-pack/agents/* .agents/agents/
cp -r typescript-pack/rules/* .agents/rules/

# Sync
hatch3r sync

# Validate
hatch3r validate
```
