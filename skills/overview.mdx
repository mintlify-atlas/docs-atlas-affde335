## What are Skills?

Skills are structured workflows that hatch3r agents follow when implementing features, fixing bugs, refactoring code, and handling other development tasks. Each skill provides:

- **Step-by-step guidance** with clear checkpoints
- **Quality gates** ensuring code meets standards before PR creation
- **Agent delegation patterns** for research, implementation, and review
- **Definition of Done** checklists for completeness
- **Related skills** for complementary workflows

## How Skills Work

### 1. Issue Type Mapping

When working on a GitHub issue, hatch3r automatically loads the appropriate skill:

| Issue Type | Skill |
|------------|-------|
| Bug report | `hatch3r-bug-fix` |
| Feature request | `hatch3r-feature` |
| Code refactor | `hatch3r-refactor` |
| Logical refactor | `hatch3r-logical-refactor` |
| Visual refactor | `hatch3r-visual-refactor` |
| QA E2E validation | `hatch3r-qa-validation` |

### 2. Agent Delegation

Skills define mandatory agent delegation points using the Task tool:

- **Researcher agents** gather context before implementation
- **Implementer agents** handle focused implementation tasks
- **Reviewer agents** perform code review before PR creation
- **Specialized agents** for audits (a11y, performance, dependencies)

### 3. Quality Gates

Every skill includes verification steps:

```bash
npm run lint && npm run typecheck && npm run test
```

Skills with UI changes add browser verification steps to visually confirm acceptance criteria.

### 4. Test-First Options

Many skills support TDD approaches:

- **Feature**: Write acceptance tests before implementing vertical slice
- **Bug Fix**: Write failing regression test before fixing
- **Refactor**: All existing tests must pass without modification

## Skill Customization

Every skill can be customized for your project:

1. Create `.hatch3r/skills/{skill-id}.customize.yaml` for structured overrides
2. Create `.hatch3r/skills/{skill-id}.customize.md` for additional workflow steps
3. Run `npx hatch3r sync` to propagate changes

See [Skill Customization](/skills/skill-customize) for details.

## Common Patterns

### Vertical Slice Delivery

Feature and bug fix skills follow a vertical slice approach:

- **Data model** changes first
- **Domain logic** implementation
- **API/backend** integration
- **UI** components and flows
- **Tests** at each layer

### Minimal Changes

Bug fix and refactor skills emphasize minimal diffs:

- Fix root cause only
- No unrelated changes
- No scope creep
- Remove dead code

### Browser Verification

Skills with UI changes include browser verification:

- Start dev server if needed
- Navigate to affected surfaces
- Confirm visual and functional correctness
- Capture screenshots for PR evidence
- Check console for errors

### Definition of Done

Every skill ends with a DoD checklist ensuring:

- Acceptance criteria met
- Tests written and passing
- Quality gates satisfied
- Documentation updated
- Security and privacy invariants respected
