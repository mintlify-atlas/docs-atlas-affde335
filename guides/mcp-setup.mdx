hatch3r ships with 5 MCP servers enabled by default and 3 opt-in servers. All secrets are centralized in a single `.env.mcp` file at the project root (gitignored by default).

## Overview

MCP (Model Context Protocol) servers provide enhanced capabilities to AI coding tools:

- **GitHub** — Access repos, issues, PRs, and projects
- **Context7** — Fetch up-to-date library documentation
- **Filesystem** — Read and write project files
- **Playwright** — Browser automation for testing
- **Brave Search** — Web research capabilities

## Where MCP Config Lives

| Tool | Config Path | When Created |
|------|-------------|-------------|
| Cursor | `.cursor/mcp.json` | `npx hatch3r init` (when MCP selected) |
| Cursor plugin | `mcp.json` (project root) | Plugin install |
| Claude Code | `.mcp.json` | `npx hatch3r init` |
| Copilot / VS Code | `.vscode/mcp.json` | `npx hatch3r init` |
| Cline / Roo | `.roo/mcp.json` | `npx hatch3r init` |

## Connecting MCP Servers

### Cursor

<Steps>
  <Step title="Run hatch3r init">
    ```bash
    npx hatch3r init
    ```
    Select MCP servers when prompted.
  </Step>
  
  <Step title="Config is written">
    - MCP config goes to `.cursor/mcp.json`
    - Secrets template goes to `.env.mcp`
  </Step>
  
  <Step title="Fill in API keys">
    Edit `.env.mcp` and add your API keys:
    ```bash
    # .env.mcp (generated by hatch3r init)
    GITHUB_PAT=ghp_xxxxxxxxxxxx
    BRAVE_API_KEY=xxxxxxxx
    ```
  </Step>
  
  <Step title="Restart Cursor">
    Restart Cursor for changes to take effect
  </Step>
  
  <Step title="Verify connection">
    In Cursor: **Settings → Tools & MCP**
    
    Verify servers show a green dot.
  </Step>
</Steps>

<Note>
  If using the Cursor plugin, the plugin provides `mcp.json` at the project root. Cursor loads project-level config from `.cursor/mcp.json` (takes precedence over global `~/.cursor/mcp.json`).
</Note>

### Claude Code

Config goes to `.mcp.json` in the project root. Fill in `.env.mcp`, source it, and restart Claude Code after init.

### Other Tools

See the [Adapters](/concepts/adapters) documentation for per-tool output paths.

## Managing Secrets

### The .env.mcp File

`hatch3r init` generates a `.env.mcp` file at the project root containing every environment variable your selected MCP servers need. **This file is covered by `.gitignore` and must never be committed.**

```bash
# .env.mcp (generated by hatch3r init)
GITHUB_PAT=ghp_xxxxxxxxxxxx
BRAVE_API_KEY=xxxxxxxx
```

When you add new MCP servers and run `hatch3r init` or `hatch3r sync`, any new variables are appended to `.env.mcp` without overwriting existing values.

### How Secrets Are Loaded Per Editor

<Tabs>
  <Tab title="VS Code / Copilot">
    **Automatic loading** — No extra steps needed.
    
    The generated `.vscode/mcp.json` includes `envFile: "${workspaceFolder}/.env.mcp"` on every STDIO server, so VS Code reads the file natively.
  </Tab>
  
  <Tab title="Cursor">
    **Source before launching:**
    
    ```bash
    set -a && source .env.mcp && set +a && cursor .
    ```
    
    **Alternative options:**
    
    - Add tokens to `~/.zshrc` / `~/.bashrc` for persistent access
    - Paste them in Cursor: **Settings → Tools & MCP → pencil icon** next to each server
  </Tab>
  
  <Tab title="Claude Code">
    **Source before launching:**
    
    ```bash
    set -a && source .env.mcp && set +a && claude
    ```
  </Tab>
  
  <Tab title="Other Editors">
    Same sourcing pattern as Cursor/Claude Code.
    
    See the [Adapters](/concepts/adapters) documentation for per-tool details.
  </Tab>
</Tabs>

## Required Environment Variables

### Default Servers

<AccordionGroup>
  <Accordion title="GitHub">
    **Environment variable:** `GITHUB_PAT`
    
    **How to get it:** [Create a Personal Access Token](https://github.com/settings/tokens/new)
    
    **Required scopes:**
    
    **Classic PAT** (Settings → Developer settings → Personal access tokens → Tokens (classic)):
    - `repo` — Full control of private repositories (read/write code, issues, PRs, projects)
    - `read:org` — Read org and team membership (needed for org projects)
    
    **Fine-grained PAT** (recommended):
    - **Repository permissions:** Contents, Issues, Pull requests, Metadata
    - **Organization permissions:** Members (read) — if using org projects
    
    Fine-grained tokens have [limitations](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens#fine-grained-personal-access-tokens-limitations) (e.g., user Projects). Classic may be required for some workflows.
    
    <Note>
      `read:packages` is only needed when running the GitHub MCP server locally via Docker. hatch3r uses the remote server, so you do not need it.
    </Note>
  </Accordion>
  
  <Accordion title="Brave Search">
    **Environment variable:** `BRAVE_API_KEY`
    
    **How to get it:** [Brave Search API](https://brave.com/search/api/)
    
    **Free tier:** 2,000 queries/month
  </Accordion>
  
  <Accordion title="Context7">
    **No secrets required.**
    
    Fetches up-to-date library documentation automatically.
  </Accordion>
  
  <Accordion title="Filesystem">
    **No secrets required.**
    
    Uses `.` (project root) as the allowed directory.
  </Accordion>
  
  <Accordion title="Playwright">
    **No secrets required.**
    
    Browser automation for testing and verification.
  </Accordion>
</AccordionGroup>

### Opt-in Servers (Enable During Init)

<AccordionGroup>
  <Accordion title="Sentry">
    **Environment variable:** `SENTRY_AUTH_TOKEN`
    
    **How to get it:** [Sentry Auth Tokens](https://sentry.io/settings/account/api/auth-tokens/)
  </Accordion>
  
  <Accordion title="Postgres">
    **Environment variable:** `POSTGRES_URL`
    
    **Format:** Your PostgreSQL connection string
    
    ```bash
    POSTGRES_URL=postgresql://user:password@host:port/database
    ```
  </Accordion>
  
  <Accordion title="Linear">
    **Environment variable:** `LINEAR_API_KEY`
    
    **How to get it:** [Linear API keys](https://linear.app/settings/api)
  </Accordion>
</AccordionGroup>

## Verifying Connection

<Steps>
  <Step title="Restart your editor">
    After setting secrets in `.env.mcp`, restart your editor
  </Step>
  
  <Step title="Check MCP status">
    **Cursor:** Settings → Tools & MCP — green dots indicate connected servers
    
    **VS Code:** Similar MCP status indicator
  </Step>
  
  <Step title="Check available tools">
    In chat/composer, check "Available Tools"
    
    You should see tools from each enabled server:
    - GitHub: `list_issues`, `create_issue`, `search_repositories`, etc.
    - Context7: `resolve-library-id`, `query-docs`
    - Brave Search: `web_search`
    - Playwright: `playwright_navigate`, `playwright_screenshot`, etc.
  </Step>
</Steps>

## Server Details

### GitHub MCP Server

- **Type:** Remote server at `https://api.githubcopilot.com/mcp/`
- **Toolsets:** Uses `X-MCP-Toolsets` header for repos, issues, pull_requests, projects
- **Authentication:** Via `GITHUB_PAT` environment variable

### Context7 MCP Server

- **Type:** Remote library documentation server
- **No authentication required**
- **Usage:**
  1. `resolve-library-id` — Find library ID (e.g., "react" → `react@18.2.0`)
  2. `query-docs` — Query documentation with natural language

### Filesystem MCP Server

- **Type:** Local filesystem access
- **Allowed directory:** `.` (project root)
- **No authentication required**

### Playwright MCP Server

- **Type:** Browser automation
- **Capabilities:**
  - Navigate to URLs
  - Click elements
  - Fill forms
  - Take screenshots
  - Execute JavaScript
- **No authentication required**

### Brave Search MCP Server

- **Type:** Web search API
- **Capabilities:**
  - Web search with natural language queries
  - Current events and documentation lookup
  - Market research and competitive analysis
- **Rate limit:** 2,000 queries/month (free tier)

## Troubleshooting

<AccordionGroup>
  <Accordion title="MCP servers not connecting">
    **Symptoms:**
    - Red or gray dots in Cursor: Settings → Tools & MCP
    - MCP tools don't appear in "Available Tools"
    
    **Solutions:**
    
    1. **Load secrets:** For Cursor, Claude Code, and most editors, source `.env.mcp` before launching:
       ```bash
       set -a && source .env.mcp && set +a && cursor .
       ```
    
    2. **Restart the editor** after running `hatch3r init` or changing MCP config
    
    3. **Verify config path:**
       - Cursor uses `.cursor/mcp.json`
       - Claude Code uses `.mcp.json`
       - See table above for other tools
  </Accordion>
  
  <Accordion title="GitHub MCP returns 401 or 403">
    **Symptoms:**
    - GitHub tools fail with authentication or permission errors
    
    **Causes:**
    - Missing or invalid `GITHUB_PAT`
    - Insufficient token scopes
    
    **Solutions:**
    
    1. Create a [Personal Access Token](https://github.com/settings/tokens/new)
    
    2. **Classic PAT:** Grant `repo` and `read:org`. For board commands, add `project`
    
    3. **Fine-grained PAT:**
       - Repository permissions: Contents, Issues, Pull requests, Metadata
       - Organization → Members (read) for org projects
       - For board commands, ensure Projects access
    
    4. Add the token to `.env.mcp`:
       ```bash
       GITHUB_PAT=ghp_xxxx
       ```
    
    5. Source `.env.mcp` and restart your editor
  </Accordion>
  
  <Accordion title="Board commands fail with permission errors">
    **Symptoms:**
    - "Permission denied" or mutation failures when using board commands
    - 403 errors when creating/updating projects or issues
    
    **Cause:**
    - The GitHub PAT lacks the `project` scope required for Projects V2 operations
    
    **Solutions:**
    
    **If using gh CLI:**
    ```bash
    gh auth refresh -s project
    ```
    
    **If using a PAT:**
    - Create or update your token to include the `project` scope (classic)
    - Or add Projects permissions (fine-grained)
  </Accordion>
</AccordionGroup>

## Per-Tool MCP Loading

hatch3r supports loading MCP servers on a per-tool basis. This allows you to:

- Use different MCP servers for different commands
- Enable expensive servers (like Brave Search) only for specific workflows
- Configure API quotas and rate limits per use case

See [Customization Guide](/guides/customization) for details on per-tool MCP configuration.
